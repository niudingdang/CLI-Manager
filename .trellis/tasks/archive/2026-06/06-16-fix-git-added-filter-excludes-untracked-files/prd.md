# 修复 Git 变更面板「新增」筛选漏掉未跟踪文件

## 问题

Git 变更面板里，未跟踪（新增）文件在「全部」视图能看到，但点击「新增」筛选标签后列表为空——尽管标签角标显示有 N 个。

## 根因

后端 `git.rs` 对未跟踪文件给出的状态是 `"U"`（`is_wt_new()` → `"U"`）。前端对「新增」存在两套不一致的定义：

- 计数 `addedCount`（`GitChangesPanel.tsx:45`）：`status ∈ {A, U, ??}`，包含未跟踪。
- 「新增」筛选按钮 value（`GitChangesPanel.tsx:51`）= `"A"`，筛选执行（`gitStore.ts:89/117`）只匹配 `status === "A"`，把 `"U"` 文件全过滤掉。

计数把 U 算进「新增」，筛选却只认 A，导致「显示有、查不出」。

## 方案

`src/stores/gitStore.ts`：抽一个匹配函数，让「新增」筛选（`A`）同时命中 `A` / `U` / `??`，与 `addedCount` 定义对齐；`fetchChanges` 与 `setStatusFilter` 两处共用该函数，消除重复表达式。

## 验收

- 「新增」筛选标签的角标计数 == 点击后列表实际显示的文件数。
- 未跟踪文件（`status === "U"`）在「新增」视图可见。
- 「全部 / 修改 / 删除」视图行为不变。

## 范围

仅改 `src/stores/gitStore.ts` 单文件，约 5 行。不改后端，不改类型，无兼容性影响。
